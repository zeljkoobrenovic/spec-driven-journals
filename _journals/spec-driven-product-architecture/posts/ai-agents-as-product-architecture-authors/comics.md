What AI agents are actually for in product architecture — in eight panels.

![Comic panel: a robot proudly presents a thick instant strategy document while a skeptical architect raises an eyebrow.](assets/images/ai-agents-as-product-architecture-authors/comic-01-instant-strategy.jpeg)
**Panel 1:** *AI agents produce plausible product prose at speed — which is both the useful part and the dangerous part.*

![Comic panel: an architect pulls a thread from a confident document and it unravels into disconnected fragments.](assets/images/ai-agents-as-product-architecture-authors/comic-02-confident-but-hollow.jpeg)
**Panel 2:** *Unconstrained, an agent builds a confident model that no evidence supports and no next session can maintain.*

![Comic panel: a robot presents a chart claiming the target customer is everyone while the architect facepalms.](assets/images/ai-agents-as-product-architecture-authors/comic-03-customers-everyone.jpeg)
**Panel 3:** *The failure modes are recognizable: generic customers, invented metrics, capabilities that are just renamed systems.*

![Comic panel: the architect hands the robot a toolbox labeled examples, skills, and validators while keeping the decision clipboard.](assets/images/ai-agents-as-product-architecture-authors/comic-04-structured-author.jpeg)
**Panel 4:** *The decision: treat agents as structured authors inside the repository — humans keep intent, judgment, and review.*

![Comic panel: a robot studies a shelf of structured domain folders and takes notes while the architect nods.](assets/images/ai-agents-as-product-architecture-authors/comic-05-inspect-the-examples.jpeg)
**Panel 5:** *Examples are authoring infrastructure: the agent starts from the current system, not from a blank prompt.*

![Comic panel: a robot carries labeled JSON blocks through a validation gate while one rejected block sits in a bin.](assets/images/ai-agents-as-product-architecture-authors/comic-06-the-validation-gate.jpeg)
**Panel 6:** *Validation changes what the agent optimizes for: the output must parse and stay consistent before anyone reviews it.*

![Comic panel: a pipeline turns a JSON source file into icons, journey panels, and a dependency map, while a disconnected doodle is crossed out.](assets/images/ai-agents-as-product-architecture-authors/comic-07-pictures-from-the-model.jpeg)
**Panel 7:** *Source-first visuals — icons, journey panels, JTBD comics, dependency maps — render the model; they never replace it.*

![Comic panel: the architect reviews a connected model board with checkmarks while the robot stands by with validated files.](assets/images/ai-agents-as-product-architecture-authors/comic-08-fast-hand-human-editor.jpeg)
**Panel 8:** *The trade: a slower structured workflow today instead of repairing a disconnected product model later — the agent is the fast hand, the human stays editor-in-chief.*
