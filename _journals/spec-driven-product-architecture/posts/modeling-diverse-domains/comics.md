Why one method models twenty different products — in eight panels.

![Comic panel: a robot unrolls a wall chart of many product-domain tiles while the architect asks why so many.](assets/images/modeling-diverse-domains/comic-01-why-so-many.jpeg)
**Panel 1:** *Ride sharing, retail, cloud, airlines, streaming, freight... the variety is not decoration.*

![Comic panel: a robot stamps identical marketplace labels onto an airplane, a server, and a pipe while the architect winces.](assets/images/modeling-diverse-domains/comic-02-one-example-narrow.jpeg)
**Panel 2:** *One kind of example teaches a narrow pattern — and the agent over-applies it everywhere.*

![Comic panel: a robot stands baffled between example boards drawn as a spiral, a pile, and a tangle of arrows.](assets/images/modeling-diverse-domains/comic-03-no-stable-model.jpeg)
**Panel 3:** *The opposite failure: if every example has its own structure, nothing transfers.*

![Comic panel: a template machine turns distinct domain shapes into identical gray blobs as the architect stops the conveyor.](assets/images/modeling-diverse-domains/comic-04-the-mush-machine.jpeg)
**Panel 4:** *The anti-pattern: forcing every domain through one template until the product story turns to mush.*

![Comic panel: a whiteboard with one fixed column of questions and three visibly different answer columns for different domains.](assets/images/modeling-diverse-domains/comic-05-same-questions-different-answers.jpeg)
**Panel 5:** *The decision: the reusable model asks the same questions; each domain must answer them differently.*

![Comic panel: a robot carries a blank domain board toward labeled archetype shelves in a small library.](assets/images/modeling-diverse-domains/comic-06-archetype-shelves.jpeg)
**Panel 6:** *How it plays out: archetypes tell a new domain which mature examples to learn structure from.*

![Comic panel: a robot reaches for a dusty old folder while the architect points a flashlight at a current, validated one.](assets/images/modeling-diverse-domains/comic-07-inspect-before-copying.jpeg)
**Panel 7:** *The cost: a living example set drifts — copying an old pattern blindly preserves the drift.*

![Comic panel: the architect and robot admire a wall of diverse domain boards with one empty slot for the next domain.](assets/images/modeling-diverse-domains/comic-08-pattern-library-grows.jpeg)
**Panel 8:** *The examples are not final truth — they are a growing pattern library for the next domain.*
