How a product stays coherent before any code exists — in eight panels.

![Comic panel: an architect surrounded by disconnected artifacts — sticky-note vision, whiteboard architecture, roadmap tool, org chart, and loose evidence papers.](assets/images/what-is-spec-driven-product-architecture/comic-01-scattered-everywhere.jpeg)
**Panel 1:** *Products lose coherence long before code appears: vision, architecture, roadmap, teams, and evidence all live in different places.*

![Comic panel: a robot prints piles of impressive but contradictory strategy documents while the architect compares two mismatched pages.](assets/images/what-is-spec-driven-product-architecture/comic-02-rex-amplifies.jpeg)
**Panel 2:** *AI agents amplify the fragmentation: plausible text everywhere, hard to compare, validate, or connect to delivery.*

![Comic panel: a robot puzzled by a one-line vision card while huge unlabeled machinery looms behind it.](assets/images/what-is-spec-driven-product-architecture/comic-03-vision-is-not-enough.jpeg)
**Panel 3:** *Vision is necessary but not operational: one sentence cannot tell an agent how jobs, pricing, payments, and support connect.*

![Comic panel: the architect presents a connected eight-layer stack from customers down to evidence while the robot admires it.](assets/images/what-is-spec-driven-product-architecture/comic-04-the-layer-stack.jpeg)
**Panel 4:** *The core move: the spec becomes a structured model of the product domain — eight connected layers, not a feature brief.*

![Comic panel: a three-station loop labeled dream, explore, and ground, walked by the architect and the robot.](assets/images/what-is-spec-driven-product-architecture/comic-05-dream-explore-ground.jpeg)
**Panel 5:** *Three modes keep the model honest: dream the intended product, explore real data, ground every concept in evidence.*

![Comic panel: the robot neatly slots labeled JSON bricks into a structured board with a validation gate, watched by the architect.](assets/images/what-is-spec-driven-product-architecture/comic-06-rex-in-the-schema.jpeg)
**Panel 6:** *With structure, examples, and validation, the agent edits source models inside a domain language — not strategy prose.*

![Comic panel: concepts tagged as evidence-backed or assumption, with the robot pointing at one drifting link.](assets/images/what-is-spec-driven-product-architecture/comic-07-visible-bets.jpeg)
**Panel 7:** *Early architecture always contains bets — grounding keeps them visible and shows when reality drifts from the dream.*

![Comic panel: the architect and the robot share one connected model board while old scattered documents sit in a recycling bin.](assets/images/what-is-spec-driven-product-architecture/comic-08-shared-language.jpeg)
**Panel 8:** *The goal: a shared product-architecture language — specific enough to be useful, structured enough to survive repeated change.*
