> **Status**: Accepted (2020-01-21)
>
> **Decision**: We will use Python as the primary programming language for the project. The work spans data analysis, machine learning, and web development, and Python is the only language in serious contention that covers all three well.

## Context

The project mixes three kinds of work: data analysis, machine learning, and a web application that exposes results to users. We need a single language across all three so the team can move data and models between contexts without rewriting code or marshalling between runtimes.

## Decision

Python will be the primary language for the project. ML and analytics code will run in the same runtime as the web layer, with native extensions used only where measured performance requires it.

## Rationale

1. **Data and ML ecosystem.** NumPy, Pandas, scikit-learn, and the broader PyData stack are the default tooling for the work we need to do. Choosing Python avoids paying a translation tax on every analysis.
2. **Readability and ramp-up.** Python's syntax is approachable, and new contributors — including ML and analytics specialists who are not full-time engineers — can be productive quickly.
3. **Community and learning material.** A large, active community means we can lean on existing examples and recipes instead of inventing patterns from scratch.
4. **Flexibility across workloads.** The same language serves prototyping, batch jobs, scientific computing, and web services, and it interoperates cleanly with C and C++ when we need to drop down.

## Implications

1. **Performance.** Python is slower than compiled languages for tight numerical loops. We will mitigate this with vectorised libraries and, where warranted, native extensions; pure-Python hot paths should be the exception, not the rule.
2. **Discipline around best practices.** Easy syntax does not mean easy systems. We will need to invest in conventions, type hints, and code review to keep larger services maintainable.
3. **Version management.** Python versions and the packaging story can be uneven. We will standardise on a supported minor version and use a single dependency-management toolchain across services.

## Conclusion

Python is the right base for a project where data analysis, machine learning, and a web application have to share code and data. The cost in raw throughput is real but manageable, and the ecosystem benefits outweigh it for our workload.
