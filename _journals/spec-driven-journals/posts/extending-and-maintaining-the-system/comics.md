How to grow a deliberately small publishing system without turning it into a hidden framework — in eight panels.

![Comic panel: a robot carries a tall stack of folders toward an engineer standing next to a small tidy machine labeled BUILD.](assets/images/extending-and-maintaining-the-system/comic-01-small-not-frozen.jpeg)
**Panel 1:** *Spec-Driven Journals is intentionally small — but it is not frozen.*

![Comic panel: a robot presents a machine overgrown with tangled pipes and levers while an engineer squints, confused.](assets/images/extending-and-maintaining-the-system/comic-02-the-hidden-framework.jpeg)
**Panel 2:** *The failure mode: extensions that turn a readable system into a hidden framework.*

![Comic panel: an engineer pins a short row of contract cards on a wall while a robot salutes them.](assets/images/extending-and-maintaining-the-system/comic-03-the-contract-wall.jpeg)
**Panel 3:** *The core idea: protect a short list of contracts — config, front matter, specs, docs output, cross-links.*

![Comic panel: a robot lifts a giant wrench labeled TEMPLATES while an engineer offers a tiny screwdriver labeled ONE POST.](assets/images/extending-and-maintaining-the-system/comic-04-smallest-change-first.jpeg)
**Panel 4:** *For each need, pick the lowest-impact change — a post before a format, a journal before a template.*

![Comic panel: a robot polishes a journal binder backstage while an engineer guards a bright doorway labeled START PAGE.](assets/images/extending-and-maintaining-the-system/comic-05-draft-before-promote.jpeg)
**Panel 5:** *A journal can build perfectly and stay off the start page — drafting and promoting are separate.*

![Comic panel: a robot tries to swap a street sign held by many chains from documents while an engineer stops him.](assets/images/extending-and-maintaining-the-system/comic-06-permalinks-are-forever.jpeg)
**Panel 6:** *Titles can change freely; a published permalink cannot — every citation makes it more expensive.*

![Comic panel: an engineer inspects a freshly printed page with a magnifying glass beside a checklist while a robot holds the next page.](assets/images/extending-and-maintaining-the-system/comic-07-build-and-inspect.jpeg)
**Panel 7:** *No test suite yet — the habit is scoped builds plus inspecting rendered pages, assets, and links.*

![Comic panel: an engineer and a robot admire a small tidy machine with one new journal shelf, contract cards straight on the wall.](assets/images/extending-and-maintaining-the-system/comic-08-a-system-you-can-read.jpeg)
**Panel 8:** *Extended, not complicated: the strength worth preserving is that a reader can understand the system.*
