# Spec: Journals as Source of Truth

## Intent

Explain the Spec-Driven Journals source-first philosophy: generated pages are useful, but durable truth lives in markdown, config, specs, and assets under version control.

## Audience

Authors, reviewers, and maintainers who need to understand where to make changes and why generated pages should not be edited directly.

## Success criteria

- [ ] Reader understands why markdown in git is canonical.
- [ ] Reader can explain why generated HTML is a review surface, not the source of truth.
- [ ] Reader sees how specs, posts, configs, and generated docs play different roles.
- [ ] Reader understands the relationship to existing foundation records.
- [ ] Reader can connect the source-of-truth principle to the official GitHub repository and generated journal URL.
- [ ] Article includes a named illustration placeholder showing source files, generated output, and the feedback loop.

## Non-goals

- Rewriting the foundation decision on markdown records.
- Tool-by-tool comparison of knowledge systems.
- Publishing governance for every journal.
