How a small sibling spec keeps every post, build, and fix pointed at the same intent — in eight panels.

![Comic panel: a robot unrolls a very long winding draft scroll across an office while an engineer inspects one end, puzzled.](assets/images/spec-driven-authoring-workflow/comic-01-the-drifting-draft.jpeg)
**Panel 1:** *The failure mode of substantial writing is not typos. It is drift.*

![Comic panel: an engineer and a robot study a detective-style wall of pages and tangled string, the robot holding a magnifying glass.](assets/images/spec-driven-authoring-workflow/comic-02-nobody-remembers-why.jpeg)
**Panel 2:** *Three sessions later, nobody can say what the piece was supposed to do.*

![Comic panel: an engineer sets a small page labeled SPEC next to a thicker page labeled POST while a robot watches.](assets/images/spec-driven-authoring-workflow/comic-03-the-sibling-spec.jpeg)
**Panel 3:** *State the intent in a sibling spec, then draft the post from that contract.*

![Comic panel: an engineer and a robot walk a circular floor path with four icon stations, the robot pushing a document cart.](assets/images/spec-driven-authoring-workflow/comic-04-the-loop.jpeg)
**Panel 4:** *Spec, draft, build, inspect, fix at source — the order is the discipline.*

![Comic panel: a small letter hops through a tiny gate labeled TYPO while a robot with a clipboard guards a tall door labeled NEW SERIES.](assets/images/spec-driven-authoring-workflow/comic-05-typos-skip-the-line.jpeg)
**Panel 5:** *A post needs a spec when the intent is not self-evident from the title.*

![Comic panel: a robot runs a printing press fed by one SPEC page, producing an article, a summary card, a script, and a comic strip.](assets/images/spec-driven-authoring-workflow/comic-06-one-spec-four-tabs.jpeg)
**Panel 6:** *The spec's Modalities section declares which docs the post warrants beyond the article.*

![Comic panel: an engineer blocks a robot's wrench from touching a rendered web page and points back at source documents on a desk.](assets/images/spec-driven-authoring-workflow/comic-07-never-patch-the-output.jpeg)
**Panel 7:** *If the page reads poorly, go back to the source and rebuild.*

![Comic panel: a robot with a clock symbol easily retrieves a paired post and spec from a tidy library shelf while an engineer smiles.](assets/images/spec-driven-authoring-workflow/comic-08-cheap-future-sessions.jpeg)
**Panel 8:** *Future sessions get cheaper: inspect the article, the spec, or both — no archaeology required.*
