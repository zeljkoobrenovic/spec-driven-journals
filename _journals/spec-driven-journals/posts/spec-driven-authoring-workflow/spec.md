# Spec: Spec-Driven Authoring Workflow

## Intent

Describe the practical workflow for creating or changing posts in Spec-Driven Journals: update the spec first, draft the post, build the journal, inspect generated output, and return fixes to source.

## Audience

Authors, reviewers, and AI-agent operators doing substantive content work in Spec-Driven Journals.

## Success criteria

- [ ] Reader can follow the source-to-spec-to-post-to-build workflow.
- [ ] Reader understands why specs come before non-trivial post edits.
- [ ] Reader knows what to verify after building.
- [ ] Reader can distinguish small edits from substantive spec-driven work.
- [ ] Reader knows the public generated journal URL to inspect after publication.
- [ ] Article includes a named illustration placeholder showing the authoring loop from spec to generated review.

## Non-goals

- Detailed editorial style guide for every journal.
- Git branching or pull-request policy.
- Instructions for generating icons or logos.
