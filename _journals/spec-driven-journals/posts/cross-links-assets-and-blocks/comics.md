How posts stay navigable while staying plain markdown — in eight panels.

**Panel 1:** *Left alone, links rot and image paths break.*

![Comic panel: a robot glues messy code scraps onto a clean page while an engineer objects.](assets/images/cross-links-assets-and-blocks/comic-02-the-paste-everything-robot.jpeg)
**Panel 2:** *The tempting fix: undocumented markup pasted into posts.*

![Comic panel: an engineer shows a robot a small toolbox with four compartments for links, assets, images, and diagrams.](assets/images/cross-links-assets-and-blocks/comic-03-small-content-model.jpeg)
**Panel 3:** *The idea: a deliberately small content model on top of markdown.*

![Comic panel: a chain connects two documents while a robot swaps one document's title placard without breaking the chain.](assets/images/cross-links-assets-and-blocks/comic-04-permalink-anchors.jpeg)
**Panel 4:** *Double-bracket links resolve by permalink; titles can change freely.*

![Comic panel: a robot points at a visible unresolved link token on a page while an engineer flags it.](assets/images/cross-links-assets-and-blocks/comic-05-visible-todo.jpeg)
**Panel 5:** *Broken links stay visible — a TODO, not a silent failure.*

![Comic panel: a robot moves framed pictures from small folders onto one shared shelf while an engineer compares matching path tags.](assets/images/cross-links-assets-and-blocks/comic-06-asset-merge.jpeg)
**Panel 6:** *Per-post assets merge at build time; the written path never changes.*

![Comic panel: a cursor clicks a small diagram and a large detailed version opens in a new window while a robot applauds.](assets/images/cross-links-assets-and-blocks/comic-07-click-to-enlarge.jpeg)
**Panel 7:** *Images render as links to themselves — click for full detail.*

![Comic panel: an engineer fits four diagram tiles into matching sockets in a text wall while blocking a robot's oversized fifth tile.](assets/images/cross-links-assets-and-blocks/comic-08-blocks-with-restraint.jpeg)
**Panel 8:** *Four block types, one fixed contract — rich content supports the argument, never replaces it.*
