> A minimal recipe for starting ADRs in a git-based project: create an adr/ directory, write one text file per decision, and commit it next to the source code.

If you use git for version control, ADRs can live in the repository alongside the code they describe. This keeps decisions discoverable, versioned, and reviewable through the same workflow as code changes.

## Minimal setup

Create a directory for ADR files:

```sh
mkdir adr
```

Create a text file for each ADR:

```sh
vi adr/database.txt
```

Write whatever you need in the ADR — templates in this repository can serve as a starting point. Then commit it:

```sh
git add adr/database.txt
git commit -m "Add ADR: database choice"
```

That's the whole loop. Reviews happen through pull requests; supersession happens by adding a new ADR and updating the old one to point to it.
