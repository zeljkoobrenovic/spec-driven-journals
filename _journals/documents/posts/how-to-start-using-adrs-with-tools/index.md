> ADRs work in whatever tool a team already uses — Google Docs, git, Jira, or a wiki. Pick the tool that matches the existing workflow rather than introducing a new one.

ADRs are a writing practice, not a tooling commitment. Use whichever tool the team already lives in:

- **Google Drive** — create a Google Doc or Google Sheet per decision; familiar editing, real-time collaboration.
- **Git** — create one file per ADR in the repository; reviewable through pull requests and versioned with the code.
- **Project tracking** — use the planning tool the team already runs in (Jira, Linear, etc.) and represent each ADR as a ticket or page.
- **Wiki** — create an ADR page per decision in MediaWiki, Confluence, Notion, or similar.

The right tool is the one the team will actually open. Introducing a new tool *just for ADRs* tends to fail; piggybacking on an existing one usually works.
