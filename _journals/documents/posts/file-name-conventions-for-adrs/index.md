> A file-naming convention for ADRs stored as text files: lowercase, dashes, present-tense imperative verb phrases, and a markdown extension.

If you keep ADRs as text files in a repository, settle on a file-naming convention before you have more than a handful — renaming later is annoying.

## Examples

- `choose-database.md`
- `format-timestamps.md`
- `manage-passwords.md`
- `handle-exceptions.md`

## Our convention

- **Present-tense imperative verb phrase.** It reads well and matches the convention we use for commit messages (`choose-database`, not `database-chosen` or `database-decision`).
- **Lowercase, hyphen-separated.** Friendly to both readers and the filesystem. No spaces, no underscores, no camelCase.
- **`.md` extension.** Markdown is plain enough to read raw and rich enough to format when needed.

That's the whole rule. Keeping it short is part of the point.
