> **Status**: Accepted
>
> **Decision**: ADRs are grouped into categories via one level of subfolder, with locally numbered ADR IDs inside each. No nested categories.

## Context

As the ADR collection grows, browsing the full list becomes harder. ADRs naturally cluster by topic (frontend, backend, data, etc.) and we want a way to express that.

## Decision Drivers

- **Easy to find groups** in collections with hundreds of ADRs.
- **Easy to group.**
- **Easy to create** a new ADR in the right place.
- **Good discoverability without external tooling.**
- **Newcomer-friendly** — a new contributor should grasp the model in under 10 minutes.
- **Template stays lean.**

## Considered Options

- Use labels (e.g. shields.io badges).
- Add `* Category: CATEGORY` under the heading.
- Use YAML front matter.
- Encode the category in the filename.
- Use subfolders with local IDs.
- Use subfolders with global IDs.
- Don't support categories.

## Decision

**Use subfolders with local IDs.** One level deep, optionally with a `to-be-categorized` folder for unclassified ADRs.

Example:

```
docs/decisions/smar/0000-secure-entities.md
docs/decisions/smar/0001-flexible-properties-selection.md
```

## Rationale

1. **Folders are natural for grouping.** Every operating system, IDE, and git client surfaces folder structure clearly.
2. **No external tooling required.** `ls`, the IDE tree view, and `find` all work out of the box.
3. **Typos are visible.** A misspelled category name shows up as a new folder, which is immediately obvious.
4. **One level only.** Restricting to a single level keeps the model simple and prevents category sprawl.

## Implications

1. **No globally unique ADR ID.** Local IDs mean `0001` exists in every subfolder. Cross-references need to include the folder.
2. **`adr-log` needs to be category-aware.** It must scan subfolders and emit per-category indexes.
3. **`e-adr` needs adjustment.** The Java annotation becomes `@ADR("category", number)` rather than `@ADR(number)`. Minor change.
4. **Unknown-category friction.** When an author doesn't know the right category, they may default to the wrong one. A `to-be-categorized` folder absorbs this case.
5. **Newcomer cost.** Categorization adds one decision before writing an ADR. For very small ADR collections, this overhead may outweigh the benefit.

## Rejected alternatives

- **Labels / badges** — full Markdown, but not parsable and not filterable in `ls`.
- **Inline `Category:` line** — parsable, but still not filterable without external tooling.
- **YAML front matter** — Jekyll-friendly, but outside CommonMark and not filterable in the filesystem.
- **Category in filename** — `ls | grep` works on Unix, but breaks down in Windows Explorer and resembles the tag-in-filename approach of TagSpaces.
- **Subfolders with global IDs** — preserves a global ID but loses the "ADR-0001 in this category" idiom.
- **No categories** — leaves large collections unmanageable.

## Conclusion

Subfolders with local IDs trade a globally unique ID for a model that works in any filesystem-aware tool, which is a worthwhile trade as collections grow.
