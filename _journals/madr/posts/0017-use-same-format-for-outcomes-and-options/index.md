> **Status**: Accepted
>
> **Decision**: The `Consequences` section uses the same `Good, because …` / `Bad, because …` format as the `Pros and Cons of the Options` section.

## Context

Earlier MADR versions used different formats for two similar things: the Decision Outcome's consequences were split into "Positive Consequences" and "Negative Consequences" subsections, while the Pros and Cons of the Options used a single list with `Good, because …` and `Bad, because …` prefixes. The mismatch made copy-and-paste between the two sections error-prone.

Ticket: [issue #75](https://github.com/adr/madr/issues/75).

## Decision Drivers

- **Consistent design of MADR.**
- **Allow easy copy and paste** between sections.

## Considered Options

- A single `Consequences` section using `Good, because …` and `Bad, because …`.
- A single `Consequences` section using `Positive, because …` and `Negative, because …`.
- No `Consequences` / `Positive Consequences` / `Negative Consequences` sections at all.

## Decision

Use a single `Consequences` section with `Good, because …` and `Bad, because …` items.

## Rationale

1. **Resolves all forces.** Matches the format used in `Pros and Cons of the Options`, making the two sections visually and lexically aligned.
2. **Copy-and-paste safe.** An argument can be moved between sections without rewording.
3. **One vocabulary to learn.** Authors use the same prefixes everywhere in the document.

## Implications

1. **Templates updated.** All MADR templates and examples use the unified format.
2. **Existing ADRs may diverge.** Older ADRs with `Positive Consequences` / `Negative Consequences` subsections are still valid; the new format is the recommendation going forward.

## Conclusion

A single consistent prefix vocabulary is cheaper to learn, write, and edit than two parallel ones.
