> **Status**: Accepted
>
> **Decision**: Markdown lists in MADR ADRs use `*` (asterisk) as the list marker, not `-` (hyphen).

## Context

Markdown allows either `*` or `-` as a list marker. The output is identical, but the source affects readability and tone.

## Considered Options

- Use an asterisk (`*`).
- Use a hyphen (`-`).

## Decision

Use the **asterisk**.

## Rationale

1. **Tone-neutral.** An asterisk does not suggest "good" or "bad". A hyphen, by contrast, can read as a minus sign — a faint negative — in contrast to a `+`. For a template that uses "Good, because…" / "Bad, because…" lines, neutrality matters.
2. **Readability.** According to the [Markdown Style Guide](http://www.cirosantilli.com/markdown-style-guide/) ([readability profile](http://www.cirosantilli.com/markdown-style-guide/#readability-profile)), the asterisk is the more readable list marker in raw text.

## Implications

1. **Templates and examples use `*`.** Any official MADR template, example, or tooling output uses the asterisk.
2. **Author convention only.** Markdown renderers do not care; this is purely a source-text convention.

## Conclusion

A trivial-seeming choice that nonetheless removes a small amount of unintended signal from the document.
