> **Status**: Accepted
>
> **Decision**: MADR's line-style headings (e.g. `Chosen option: "..."`) are written in plain Markdown. They are not emphasized with bold or italics.

## Context

MADR contains lines such as `Chosen option: "[option 1]"`. The question is whether the leading label ("Chosen option") should be emphasized — for example bolded as `**Chosen option**: "[option 1]"`.

## Decision Drivers

- **MADR should be easy to read.**
- **MADR should be easy to write.**

## Considered Options

- Do not emphasize line headings.
- Emphasize line headings (e.g. with bold).

## Decision

**Do not emphasize line headings.**

## Rationale

1. **Position already signals role.** These labels appear at the start of a line and are followed by a colon. That is enough visual structure to identify them.
2. **Stars confuse newcomers.** Readers unfamiliar with Markdown can find `**...**` distracting in the raw source.

## Implications

1. **Simpler source.** Authors write plain text without remembering an emphasis convention.
2. **Consistent rendering.** Output looks the same across renderers that handle emphasis differently.

## Conclusion

The line-and-colon convention is enough on its own. Adding emphasis would not improve readability and would make raw Markdown harder to read for beginners.
