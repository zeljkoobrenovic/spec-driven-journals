> **Status**: Accepted
>
> **Decision**: The first heading of an ADR contains the title only. The ADR number is encoded in the filename, not in the heading.

## Context

ADRs need a unique identifier. The question is where to put it: in the heading (`# 2. Do not use numbers in headings`) or only in the filename (`0002-do-not-use-numbers-in-headings.md`).

Options:

- Use the title only in the heading.
- Add the ADR number in front of the title.

## Decision

Use the **title only** in the heading. The ADR number lives in the filename.

## Rationale

1. **Consistent with Markdown convention.** In most Markdown documents the heading is the title, and numbering is applied by the renderer or CSS — not inserted by hand.
2. **Easier renaming.** ADRs (before publication) can be renumbered without rewriting the heading.
3. **Portable across repositories.** An ADR can be copied between repositories without renumbering the body.

## Implications

1. **TOC tooling reads the number from the filename**, not the body.
2. **The heading stays a clean human-readable title.**
3. **Number changes are filename-only**, which keeps Git history of the content intact.

## Conclusion

Filenames carry numbers; headings carry titles. Each piece does the job it is good at.
