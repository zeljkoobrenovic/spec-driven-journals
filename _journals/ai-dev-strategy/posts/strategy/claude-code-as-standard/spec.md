# Spec: Why Claude Code — The Capabilities That Matter

> Filled in during the second pass over backfilled specs. Status `draft`.

## Intent

Explain why Claude Code is the 2026 default: **how its capabilities
combine** (agentic loop + subagents + MCP + CLAUDE.md + Skills +
hooks + settings) into one governed modernization surface. Make clear
the case rests on **combination**, not on uniqueness of any single
primitive. Strategic assets remain **understandable and transferable**:
markdown, scripts, templates.

## Audience

- **Engineers** new to Claude Code.
- **Architecture group** evaluating the tooling argument.
- **Future selves** revisiting after the default is challenged.

## Success criteria

- [ ] Reader can name the primitives.
- [ ] Reader knows the argument is about **combination**, not
      uniqueness.
- [ ] Reader knows the strategic assets are **transferable**.

## Non-goals

- Defining Claude Code primitives in detail (that's
  [[claude-md-and-memory]] and [[mcp-skills-subagents]]).
- Claiming permanent superiority.

## Open questions

- How to communicate the transfer story to migrate-to-X readers.

## Decision log

- **2026-05-20** — Framed argument as **combination**, not
  primitive uniqueness. Considered single-primitive arguments;
  rejected because they invite "competitor has this too" rebuttals.
