# Spec: Reference — MCP, Skills, and Subagent Details

> Filled in during the second pass over backfilled specs. Status `draft`.

## Intent

Provide **detailed reference material** for MCP servers, Skills, and
subagents — how agentic tools reach systems, apply internal
conventions, and run repeatable modernization workflows. This is
reference; the main reading path lives in [[claude-code-enablement]].

## Audience

- **Engineers** building Skills or subagents.
- **AI Engineering Enablement** registering MCP servers.
- **Power users** customising workflows.

## Success criteria

- [ ] Reader knows the distinct purposes of MCP / Skills / subagents.
- [ ] Reader can register a new capability in the right slot.
- [ ] Reader knows this is **reference**, not the main path.

## Non-goals

- Listing every MCP server or Skill.
- Replacing the main enablement narrative.

## Open questions

- How registries handle naming collisions and versioning.
